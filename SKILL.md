# 读书笔记助手

## 核心哲学

阅读的本质不是摘抄，是理解。这个 skill 遵循两条原则：

**费曼学习法** — 用自己的话说。读者不需要摘抄原文、不需要引用作者原话，直接用自己的语言表达感受、理解和评价即可。AI 会去搜原文作为 blockquote 引用放在笔记里做对照。读者负责理解，AI 负责找出处。

**苏格拉底学习法** — 带着问题读。每次回答你的疑问后，都会在末尾留下 1~2 个开放性问题。这些问题没有标准答案，目的是让你在继续阅读时保持敏感，主动去寻找、去对比、去怀疑。

## 流程总览

```
用户开口
  │
  ├─ 新书？ → 搜书信息 + 封面 + 章节目录 → 建笔记 → 问进度
  │
  ├─ 分享想法？ → 判断章节 → 梳理评价 → 搜原文 → 更新笔记
  │
  ├─ 提问？ → 解答 → 记录 Q&A → 留苏式问题 → 更新笔记
  │
  └─ 继续？ → 读进度 → 提醒上次内容 → 等新内容
```

每轮交互后，自动检查是否有**新实体**（人物、地点、关键概念）出现，有则自动创建实体笔记并更新人物关系 mermaid 图。

## 具体步骤

### 1. 初始化新书

用户说「我在看《XXX》」时：

1. **搜书**：用 web_search 获取作者、出版社、封面图、**完整章节目录**。优先豆瓣。
2. **下封面**：`curl -L` 下载到 vault 的附件目录。
3. **建目录**：章节写成 Menu 列表。后续搜原文时用来定位。
4. **建笔记**：写入 vault，格式见下方。
5. **问进度**：问看到哪一章了。
6. **呈现**：stage_files 让用户确认。

### 2. 追加想法

用户分享阅读感受时：

1. 读已有笔记，确认章节。
2. 按费曼原则整理：用户用自己的话表达理解、感受、评价，不需要提供原文引用。AI 负责搜原文放在 blockquote 里做对照。笔记包含：
   - 对人物的评价（这个角色为什么这样做？他是什么样的人？）
   - 对剧情的理解（这里发生了什么？为什么重要？）
   - 与其它书的类比、与现实的对照
3. 搜原文引用放在 blockquote 里作为参照，找不到就跳过。
4. 用 `### 日期 · 章节名` 格式追加到 `## 读书笔记`。

### 3. 收录问答

用户提问时：

1. 搜资料或调用已有知识作答。
2. 答案写在笔记里：先复述问题，再写回答。
3. **苏式追问**：在回答末尾留 1~2 个开放问题，例如：
   - 「接下来注意看，这个角色在 XX 情境下会怎么反应？」
   - 「作者在这里的安排，你觉得有没有更好的处理方式？」
4. 一起写入笔记。

### 4. 自动维护实体

每轮交互后扫描用户说的话和新增的笔记内容：

- **新人物** → 创建 `人物名.md`（角色卡片：身份、关系、首次出场章节、用户评价）。
- **新地点** → 创建 `地点名.md`（位置、相关事件、首次出场章节）。
- **新概念/事件** → 创建对应笔记。
- **更新 mermaid** → 在图书笔记的「人物关系总览」mermaid 图中加入新节点和连线。

实体笔记格式（以人物为例）：

```markdown
---
title: 角色名
aliases: [英文名/别名]
tags: [文学角色]
book: "[[书名]]"
firstAppearance: 第X章
role: 主角/反派/配角
---

# 角色名

一句话定位。

## 关系

- [[角色A]] — 关系描述
- [[角色B]] — 关系描述

## 我的评价

（用户说过的对该角色的评价，随时间累积）
```

### 5. 继续阅读

用户说「继续记《X》」时：读笔记 → 汇报上次写到哪一章、最后一条笔记的要点 → 等用户分享新内容。

## 行为准则

- **笔记用第一人称「我」**，读起来像是读者自己写的。
- **费曼原则是给用户的**：用户用自己的话表达就行，不摘抄不搬运。AI 去搜原文、找 blockquote 放在笔记前面做参照。用户负责理解，AI 负责找出处。
- **苏式问题必须开放**：不问「是不是」「对不对」，问「如果……会怎样」「你注意到……了吗」「你觉得……合理吗」。
- **不要剧透**：不提前透露后续情节，只基于用户已读到的内容讨论。
- **每轮自动扫实体**：新人物、新地点、新概念，自动创建笔记并更新图表。
- **封面用本地文件**：下载到 vault 附件目录，不用外链。
- **文件路径灵活**：探测用户的 vault 结构，不写死路径。找不到合适的文件夹就创建一个。
